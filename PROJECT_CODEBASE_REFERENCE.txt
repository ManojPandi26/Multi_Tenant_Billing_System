ARCHITECTURE OVERVIEW:
  - Schema-per-tenant PostgreSQL isolation via Hibernate SCHEMA multi-tenancy
  - Two billing domains: platform billing (tenant pays platform) and
    business billing (tenant bills their own customers)
  - Dual auth surfaces: per-tenant JWT (cookie + refresh token) and
    platform admin auth (Redis-backed refresh tokens)
  - Razorpay as payment processor (INR currency)
  - Quartz scheduler for background jobs across all tenant schemas
  - Flyway migrations split across public/ (platform) and tenant/ (per-tenant)

    itemTax = ROUND((unitPrice * qty) * (taxPct / 100), 2)
    itemTotal = ROUND((unitPrice * qty) + itemTax, 2)
    invoice.subtotal = SUM(unitPrice * qty)
    invoice.taxAmount = SUM(itemTax)
    invoice.totalAmount = subtotal + taxAmount
  KEY METHODS:
    - create(CreateBusinessInvoiceRequest) - Creates DRAFT with line items
    - getById(Long) - Get by ID
    - list(Pageable) - Paginated list
    - addLineItem(Long invoiceId, AddLineItemRequest) - Add line item to DRAFT,
      recalculates totals from DB
    - removeLineItem(Long invoiceId, Long itemId) - Remove from DRAFT,
      recalculates totals from DB
    - finalize(Long invoiceId) - DRAFT -> OPEN, sets 30-day due date
    - send(Long invoiceId) - Sends invoice email with PDF attachment,
      fires BUSINESS_INVOICE_SENT event
    - voidInvoice(Long invoiceId) - Voids DRAFT/OPEN invoice
    - markPaid(Long invoiceId) - Called by payment service when fully paid
    - createPaymentLink(Long invoiceId) - Creates Razorpay Payment Link (idempotent)
    - generatePdf(Long invoiceId) - PDF generation via BusinessInvoicePdfService
    - generateInvoiceNumber() - BINV-{tenantId}-{YYYYMM}-{seq:04d}
  EVENTS: Publishes BUSINESS_INVOICE_SENT via BusinessEventPublisher
