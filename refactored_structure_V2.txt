================================================================================
                     MULTITENANT BILLING SYSTEM
                     REFACTORED ENTERPRISE PROJECT STRUCTURE
================================================================================

Project     : MultiTenantBillingSystem
Package     : com.mtbs
Java Ver.   : 17
Spring Boot  : 3.4.3
Build Tool   : Maven (Multi-Module)
Database     : PostgreSQL (schema-per-tenant multi-tenancy)

================================================================================
                        MAVEN MODULE LAYOUT
================================================================================

MultiTenantBillingSystem/               [Root POM]
├── mtbs-shared-kernel/                 [Shared across all modules — zero deps on others]
├── mtbs-tenant-core/                   [Tenant lifecycle, onboarding, plan management]

├── mtbs-notification/                  [Email, SMS, webhook notifications]
├── mtbs-admin/                         [Super-admin, platform metrics, cross-tenant ops]
└── mtbs-app/                           [Spring Boot entry point, wires all modules]

================================================================================
                         KEY ARCHITECTURAL CHANGES SUMMARY
================================================================================

CHANGE                          REASON
──────────────────────────────────────────────────────────────────────────────
mtbs-business: vertical slice   Replaced horizontal layers (entity/repo/service/
(feature-based packaging)       controller) with feature-based sub-packages
                                (customer/ product/ invoice/ payment/ report/).
                                Each feature owns its full stack — zero cross-
                                feature class dependencies. New features added
                                without touching existing packages. Aligns with
                                DDD Aggregate boundaries and enables eventual
                                microservice extraction per feature.

Maven multi-module              Enforces bounded context boundaries,
                                independent builds, path to microservices

TenantAwareTaskDecorator        Fixes ThreadLocal blindspot in @Async,
                                CompletableFuture, virtual threads

Transactional Outbox Pattern    Eliminates dual-write inconsistency between
(outbox_events table)           DB commit and event publish

Quartz to PUBLIC schema         Removes QRTZ_* table explosion (was N tenants
                                × 11 tables); single shared Quartz JDBC store

NotificationPort interface      Decouples email/SMS/webhook behind a port;
                                swap or add providers without core changes

gateway_metadata JSONB on       Future-proofs multi-gateway support (Stripe,
payments table                  PayPal) without schema changes

CQRS read models                Separate read-path for dashboards and reports;
                                can point to read replica for heavy queries

AuditLog entity (PUBLIC)        Immutable audit trail — who did what, when;
                                required for billing compliance

MapStruct mappers               Replaces manual mapping; eliminates PlanMapper
                                misplaced in dto/; compile-time safety

ArchUnit boundary tests         Enforces that modules don't cross-import;
                                prevents ball-of-mud regression

Testcontainers integration      Real PostgreSQL in CI; actual schema switching
tests                           tested, not mocked
